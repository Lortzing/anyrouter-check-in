# Project Agent Rules

- Work from the repository root and read this file before modifying the project.
- Use the project-local `uv` environment for Python commands and dependencies.
- Preserve existing check-in behavior unless a task explicitly requires logic changes.
- Keep notification errors account-specific and retain detailed failures in GitHub Actions logs.
- Run at least a syntax check or relevant tests after code changes.
- Update this file when project-wide development rules change.
